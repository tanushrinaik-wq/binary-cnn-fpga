Project Description:

EdgeProbe is an investigative Edge AI project focused on studying whether Binary Convolutional Neural Networks (BCNNs) can provide a practical efficiency advantage for constrained edge-style inference when compared with conventional floating-point software execution. Instead of claiming a completed on-board deployment, this project evaluates the idea through a combined software-training, export, and RTL-simulation workflow, with emphasis on model compression, binary arithmetic, and latency-oriented analysis.

The workflow starts with the Rock-Paper-Scissors dataset, which is remapped into a binary classification task and used to train a compact BCNN student model with Straight-Through Estimator (STE) based binarization and Knowledge Distillation from a MobileNetV2 teacher network. The trained binary parameters are then exported into Verilog-compatible `.hex` and `.mif` files, along with software reference artifacts used to align the machine-learning pipeline with the RTL inference flow.

The hardware side of the project is framed as a simulation-first study targeting a Cyclone IV style accelerator architecture rather than a completed FPGA board deployment. The RTL path models SPI-based image ingestion, pixel binarization, XNOR-popcount convolution, pooling, and final classification in ModelSim. The final analysis is based on generated simulation artifacts and CPU timing comparisons, including float Keras inference versus software binary inference, to investigate the expected edge-efficiency benefits and practical design trade-offs.
